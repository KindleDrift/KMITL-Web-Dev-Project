# KMITL-Web-Dev-Project

An ASP.NET Core MVC Web Application Project as a part of Web Application Development coursework at KMITL.

This repository contains the source code for a simple web board app.

## Getting Started
### Prerequisites
- [.NET 10.0 SDK](https://dotnet.microsoft.com/download)
- Visual Studio 2022 or later. (Alternatively, editor of your choice.)

### Installation
1. Clone the repository:
   ```sh
   git clone https://github.com/KindleDrift/KMITL-Web-Dev-Project.git
   ```
2. Open the solution file `WebDevProject.slnx` in your IDE.
3. Restore dependencies:
   ```sh
   dotnet restore # If using Visual Studio, NuGet should automatically install the dependencies by default.
   ```
4. Build and run the project:
   ```sh
   dotnet run --project WebDevProject/WebDevProject.csproj # Or click the run button in Visual Studio.
   ```

### Database Configuration

This document explains how to set up and connect to the local SQL Server database (`localdb`) which is already configured for this development environment.

#### 1. Verify LocalDB is Installed

You can check if LocalDB is installed and running on your machine by opening a terminal and running:

```bash
sqllocaldb info
```

You should see an output that includes `MSSQLLocalDB`.

#### 2. Check the Configuration

The application is already configured to use the LocalDB instance. You can verify this in the following files:

**`WebDevProject/appsettings.json`**:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=Db;Trusted_Connection=true;MultipleActiveResultSets=true;"
  }
}
```

**`WebDevProject/Program.cs`**:
```csharp
// The provider is already set to SQL Server
builder.Services.AddDbContext<ApplicationDbContext>(options =>
    options.UseSqlServer(connectionString));
```
*(No changes are needed here if you are using LocalDB).*

#### 3. Apply Database Migrations

To create the database and required tables, you need to apply Entity Framework Core migrations.

1. **Install the EF Core CLI Tool** (if you haven't already globally installed it):
   ```bash
   dotnet tool install --global dotnet-ef
   ```

2. **Create the Initial Migration**:
   Run this from the root of the project to generate the migration files:
   ```bash
   dotnet ef migrations add InitialCreate --project WebDevProject
   ```

3. **Update the Database**:
   Apply the migration to the LocalDB instance:
   ```bash
   dotnet ef database update --project WebDevProject
   ```

After these steps, the local database will be fully set up and ready to use!

#### Extra note on database

On the first run, the database will be seeded by the `DbSeeder` file.
It will generate a default admin user and 26 users. See `DbSeeder.cs` for user details. Non of the users will have their onboarding completed, so on signing in, they will be redirected to the onboarding page.
There will be a few boards and posts generated by the `DbSeeder` file. (Note that author of board will not see their own boards in search results)

## Features
- User authentication (sign up, sign in, sign out, password changes)
- Custom onboarding flow for new users
- Board/Event management (create, read, update, delete boards)
- Board membership system (join, leave, approve, reject participants)
- User notification system alerts
- User profiles with profile picture uploading
- Admin dashboard for managing operations
- Automated database seeding for easy development setup
- Time zone handling for accurate event display

## Folder Structure
```
WebDevProject/
├── Controllers/         # MVC controllers
├── Models/              # MVC models
├── Views/               # MVC views
├── Filters/             # Filters for page redirects (Only Onboarding)
├── Helpers/             # For custom helper methods (Only Timezone conversion)
├── Services/            # For custom services (Mainly reused methods for controllers)
├── wwwroot/             # Static files (CSS, JS)
├── Properties/          # Project properties
├── appsettings.json     # Configuration files
├── Program.cs           # Entry point
└── WebDevProject.csproj # Project file
```

## Page Routes
- `/` or `/Home`: Landing/Home page
- `/Account/SignUp`: User registration
- `/Account/SignIn`: User login
- `/Account/Onboarding`: Required profile setup for new users
- `/Board`: View all boards/events
- `/Board/Create`: Create a new board
- `/Board/Details/{id}`: View specific board details and manage participants
- `/Profile`: View your user profile
- `/Profile/Edit`: Edit user details and profile picture
- `/Notifications`: View user notifications
- `/Admin`: Admin-specific management routes
